# AI Tool Brief Examples

Use these examples when the product is an AI tool and the brief still sounds too generic.

The goal is not to copy a category. The goal is to make the workflow, input, output, and trust cues concrete.

## Example 1: AI Character Generator

### Product

- Name: CharGen
- One-line positioning: AI character generator for game masters, writers, and indie studios

### Audience

- Primary audience: tabletop game masters and fiction creators
- Secondary audience: indie game teams and hobby artists
- Usage context: rapid concepting, campaign prep, and reusable character asset creation

### Goal

- Primary conversion: generate a first character
- Secondary conversion: upgrade for more generations and export options

### AI Product Workflow

- Core workflow: user writes a short character concept, selects a style, and gets a portrait plus structured backstory
- User input type: text prompt and style selection
- System output type: image plus name, role, traits, and backstory card
- Demo or preview expectation: hero should show prompt input on the left and generated character card on the right
- Trust proof for output quality: sample outputs across fantasy, sci-fi, and anime
- What makes this AI tool different from adjacent tools: combines visual output and structured worldbuilding instead of only image generation

## Example 2: AI Meeting Notes Tool

### Product

- Name: Briefly
- One-line positioning: AI meeting notes and action-item capture for product and operations teams

### Audience

- Primary audience: product managers, founders, operations leads
- Secondary audience: agencies and client-facing teams
- Usage context: internal meetings, customer calls, project syncs

### Goal

- Primary conversion: upload or paste a meeting transcript
- Secondary conversion: start a free workspace

### AI Product Workflow

- Core workflow: user uploads a recording or transcript, selects note style, and gets structured notes with decisions and action items
- User input type: transcript, audio file, or meeting text
- System output type: summary, decisions, action items, and follow-up draft
- Demo or preview expectation: show raw transcript snippet transforming into structured notes
- Trust proof for output quality: sample before/after, support for speaker detection, visible action-item extraction
- What makes this AI tool different from adjacent tools: optimized for operational follow-through, not only summarization

## Example 3: AI Screenshot to UI Tool

### Product

- Name: FrameLift
- One-line positioning: Turn rough screenshots and hand-drawn wireframes into editable UI concepts

### Audience

- Primary audience: indie makers, designers, product teams
- Secondary audience: agencies validating page ideas quickly
- Usage context: early concepting, redesign exploration, team reviews

### Goal

- Primary conversion: upload a screenshot or sketch
- Secondary conversion: create a project

### AI Product Workflow

- Core workflow: user uploads an image, chooses a design direction, and gets an editable UI concept with layout suggestions
- User input type: screenshot, sketch, or wireframe image
- System output type: styled UI mockup and editable layout concept
- Demo or preview expectation: strong before/after visual in hero
- Trust proof for output quality: multiple source types, editable results, layout fidelity cues
- What makes this AI tool different from adjacent tools: preserves structure from the source instead of generating unrelated UI

## Example 4: AI SEO Page Generator

### Product

- Name: RankDraft
- One-line positioning: Generate search-intent-aligned landing pages from product data and keyword targets

### Audience

- Primary audience: SaaS founders, growth marketers, SEO operators
- Secondary audience: agencies producing large numbers of programmatic pages
- Usage context: SEO page production, campaign landing pages, long-tail expansion

### Goal

- Primary conversion: generate a first landing page draft
- Secondary conversion: connect data source or start a workspace

### AI Product Workflow

- Core workflow: user provides keyword, page type, and product details, then receives a structured landing page draft
- User input type: keyword target, product info, page goal
- System output type: page outline, draft copy, and recommended structure
- Demo or preview expectation: show keyword and product input becoming a page draft preview
- Trust proof for output quality: SERP-aligned headings, SEO checklist, editable structure
- What makes this AI tool different from adjacent tools: focused on page-level conversion plus search intent, not generic AI writing

## Reusable Patterns

When writing briefs for AI tools, prefer these patterns:

- Show `input -> output` clearly in the hero
- Use one believable demo, not five abstract features
- Name the workflow in plain language
- Make trust visible: quality, speed, privacy, accuracy, editing control, or examples
- Differentiate the product from adjacent AI tools in one concrete sentence

Avoid these patterns:

- "AI-powered" with no visible workflow
- abstract feature lists with no demo
- hero sections that say what the product promises but not what it actually does
- generic claims like smarter, faster, seamless, revolutionary
