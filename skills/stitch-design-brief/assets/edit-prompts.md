# Stitch Edit Prompt Starters

Use these as targeted follow-ups after a first-generation screen.

## 1. Strengthen SEO Structure

```text
Refine this page to improve SEO structure and search-intent clarity without changing the overall visual direction.

Improve:
- the H1 so it clearly matches the main topic
- H2 section headings so they sound like real search-intent topics
- scenario entry cards so they are explicit and indexable
- resource card titles and descriptions so they include practical use case details
- FAQ so it answers realistic search questions

Avoid keyword stuffing and avoid generic startup language.
```

## 2. Strengthen Conversion

```text
Refine this page to make the primary CTA more obvious and conversion-oriented.

Improve:
- hero hierarchy
- CTA prominence
- proof and trust around the main offer
- clarity around what is free, downloadable, or previewable

Do not change the overall brand tone.
```

## 3. Reduce AI-Looking Design Patterns

```text
Refine this page to feel less generic and less AI-generated.

Reduce:
- repetitive equal-sized card grids
- centered everything
- generic gradient styling
- vague feature marketing blocks

Increase:
- editorial hierarchy
- asymmetry
- clearer product specificity
- more grounded resource previews
```

## 4. Increase Buyer Trust

```text
Refine this page so it feels more credible for the primary buyer persona and decision-maker.

Increase:
- trust signals
- practical language
- scenario realism
- operational clarity

Use the actual target audience and buying context of the page.
This prompt is especially useful for B2B, service, enterprise, or high-trust conversion pages.

Avoid making it colder or more corporate-template-looking unless the brief explicitly calls for that.
```

## 5. Rewrite Generic Marketing Copy

```text
Refine the copy on this page so it sounds like it was written by a human who understands the product and its real use cases.

Replace:
- vague outcome statements such as unlock, empower, transform, seamlessly
- generic taglines that could apply to any product
- passive feature blurbs with little scenario grounding

Replace them with:
- specific, scenario-based language
- direct, practical wording
- more concrete user outcomes
- clearer CTA labels and section intros

Do not make the tone flat or robotic. Keep the current visual direction and brand warmth.
```

## 6. Clarify AI Tool Value Proposition

```text
Refine this page so the AI tool's value proposition is concrete, specific, and demonstrable.

Improve:
- what the user inputs
- what the tool outputs
- what a realistic result looks like
- where the demo, preview, or sample output appears on the page
- why a user should trust the output quality

Replace vague "AI-powered" framing with:
- a clearer workflow
- more explicit before/after or input/output states
- stronger product-specific differentiation

Do not turn the page into a technical docs page. Keep it visually polished and conversion-oriented.
```
