# Product Thinking Router Scenarios

## 1. Should we build this feature?
- Primary category: Prioritization
- Secondary category: Discovery
- Recommended frameworks:
  - JTBD
  - Kano
  - RICE
- Output goal:
  - Clarify the real job behind the request
  - Understand satisfaction impact
  - Decide whether it deserves investment now

## 2. Users keep asking for this
- Primary category: Discovery
- Secondary category: Prioritization
- Recommended frameworks:
  - JTBD
  - Kano
  - User Journey Map
- Output goal:
  - Separate surface requests from real needs
  - See whether the request reflects a broken journey or unmet expectation

## 3. I have a new product idea. How do I structure it?
- Primary category: Strategy
- Secondary category: Discovery
- Recommended frameworks:
  - Lean Canvas
  - Assumption Tree
  - JTBD
- Output goal:
  - Define problem, user, and value proposition
  - Identify the biggest assumptions to validate first

## 4. Growth is slowing
- Primary category: Growth
- Secondary category: Diagnosis
- Recommended frameworks:
  - AARRR
  - User Journey Map
  - 5 Whys
- Output goal:
  - Identify the weakest funnel stage
  - Find likely friction and root causes

## 5. How do I prioritize backlog or roadmap?
- Primary category: Prioritization
- Secondary category: Planning
- Recommended frameworks:
  - RICE
  - MoSCoW
  - ICE
- Output goal:
  - Rank opportunities
  - Define release boundaries

## 6. What should our North Star Metric be?
- Primary category: Metrics
- Secondary category: Strategy
- Recommended frameworks:
  - North Star Metric
  - HEART
- Output goal:
  - Define the top-level value metric
  - Add experience guardrails or supporting metrics

## 7. Why is retention low?
- Primary category: Growth
- Secondary category: Experience
- Recommended frameworks:
  - AARRR
  - User Journey Map
  - HEART
- Output goal:
  - Locate the retention drop
  - Connect it to experience friction

## 8. What should MVP include?
- Primary category: Planning
- Secondary category: Prioritization
- Recommended frameworks:
  - MoSCoW
  - ICE
  - Lean Canvas
- Output goal:
  - Reduce scope
  - Keep only what supports the key hypothesis

## 9. Competitor launched this feature. Should we follow?
- Primary category: Prioritization
- Secondary category: Discovery
- Recommended frameworks:
  - JTBD
  - Kano
  - RICE
- Output goal:
  - Avoid reactive copying
  - Check whether the feature maps to real user value and current priority

## 10. We shipped it, but adoption is weak
- Primary category: Diagnosis
- Secondary category: Review
- Recommended frameworks:
  - 5 Whys
  - HEART
  - AARRR
- Output goal:
  - Separate awareness, activation, experience, and retention causes
