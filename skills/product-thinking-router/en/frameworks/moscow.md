# MoSCoW

## What problem it solves
MoSCoW helps define scope boundaries and align stakeholders on what must be included versus what can wait.

## Best for
- MVP scoping
- release planning
- stakeholder negotiation around scope

## Not for
- deep discovery of user motivation
- quantitative prioritization across many options

## Use when
- the core issue is scope control
- the team needs a release boundary
- “everything is important” is creating delivery risk

## Categories
- Must have
- Should have
- Could have
- Won’t have for now

## Output template
Item: ______
Category: ______
Reason: ______
Risk if excluded: ______
Review point: ______

## Common misuse
Do not label half the backlog as “Must have.”
If everything is must-have, the model has failed.
