# Kano Model

## What problem it solves
Kano helps classify features by how they affect satisfaction.

## Best for
- deciding whether something is basic, performance, or delight
- understanding whether a feature is table stakes or differentiator

## Not for
- direct roadmap scoring
- ROI comparison by itself

## Use when
- the debate is about customer expectation and satisfaction impact
- you need to know whether a feature is must-have or nice-to-have

## Core categories
- Must-have
- Performance
- Delighter
- Indifferent
- Reverse

## Output template
Feature: ______
Likely category: ______
Reason: ______
If missing, user reaction: ______
If improved, user reaction: ______

## Common misuse
Do not use Kano alone to rank a backlog.
A delighter with low reach and high cost may still rank below a must-have issue.
