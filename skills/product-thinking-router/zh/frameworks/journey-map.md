# User Journey Map（用户旅程地图）

## 一句话定义

User Journey Map 揭示用户在完整体验流程中的摩擦点、期望落差和流失节点。

---

## 解决什么问题

当产品体验感觉"碎片化"，或者用户在多个触点都有问题时，很难从单点分析找到根因。User Journey Map 帮助你从用户视角，把整个体验流程可视化，找到最关键的摩擦点。

**从什么状态到什么状态**：
- 从：知道有问题，但不知道问题在哪个环节
- 到：清楚看到每个阶段的用户目标、行为、痛点和机会

---

## 适用场景

- **场景 1**：产品体验感觉断裂，用户在多个地方卡住
- **场景 2**：激活率低，需要找到新用户流失的具体节点
- **场景 3**：团队对"用户体验"有分歧，需要共同视角

---

## 不适用场景

- **不适用于**：高层商业战略决策（原因：Journey Map 关注体验细节，不适合宏观战略）
- **不适用于**：跨主题的数字化优先级比较（原因：Journey Map 是体验工具，不是 ROI 工具）

---

## 输入要求

**必须知道的**：
- 要分析的用户旅程范围（如：新用户激活、核心功能使用）
- 用户的核心目标

**有助于但非必须的**：
- 用户访谈或调研数据
- 产品数据（各步骤转化率、流失点）
- 用户反馈和抱怨

---

## 输出模板

```
旅程范围：______（如：从注册到完成第一个核心任务）
用户类型：______

阶段 1：______
  用户目标：______
  用户行为：______
  痛点：______
  情绪 / 期望：______
  机会点：______

阶段 2：______
  用户目标：______
  用户行为：______
  痛点：______
  情绪 / 期望：______
  机会点：______

（继续各阶段...）

最关键的摩擦点：______
最高优先级的机会：______
```

---

## 常见误用

> ⚠️ 最常见的误用：画了漂亮的旅程图，但没有找到可执行的改进点
>
> 错误做法：花大量时间制作精美的旅程图，但结论只是"用户体验需要改善"
>
> 为什么错误：Journey Map 的价值在于找到具体的摩擦点和机会，而不是展示
>
> 正确做法：每个阶段必须识别出具体的痛点，并给出可执行的机会点

---

## 真实案例

**场景**：SaaS 产品注册量很高，但 7 日留存只有 20%

**旅程分析（新用户激活）**：

阶段 1：注册
- 用户目标：快速开始使用
- 用户行为：填写邮箱、设置密码
- 痛点：需要验证邮箱，等待时间长
- 情绪：期待 → 轻微不耐烦
- 机会：简化注册流程，支持 Google 登录

阶段 2：首次进入产品
- 用户目标：理解产品能做什么
- 用户行为：浏览界面，不知道从哪里开始
- 痛点：没有引导，界面信息量大
- 情绪：困惑
- 机会：增加新手引导，突出核心功能入口

阶段 3：尝试核心功能
- 用户目标：完成第一个有价值的任务
- 用户行为：摸索功能，多次点击错误
- 痛点：核心功能需要 5 步才能完成
- 情绪：挫败
- 机会：简化核心任务流程，减少步骤

**最关键的摩擦点**：阶段 2（首次进入）和阶段 3（核心功能）
**最高优先级的机会**：增加新手引导 + 简化核心任务流程

## 工具组合说明

| 搭配工具 | 组合逻辑 |
|---------|---------|
| AARRR | AARRR 定位漏斗断点，Journey Map 找到具体摩擦 |
| 5 Whys | Journey Map 找到摩擦点，5 Whys 挖掘根因 |
| HEART | Journey Map 找摩擦，HEART 量化体验问题 |

---

## 关键问题清单

使用 User Journey Map 时，必须回答：

1. 要分析的旅程范围是什么？
2. 每个阶段用户的真实目标是什么？
3. 哪个阶段的摩擦最严重？
4. 摩擦背后的原因是什么？
5. 最高优先级的改进机会是什么？
